# Salon LP Builder

HPBサロン掲載ページから、CVR最大化LPの7セクション構成コピーを自動生成するスキル。

## 起動条件

以下のいずれかで起動：
- ユーザーが `beauty.hotpepper.jp/.../slnH......` のURLを送信
- 「サロン分析してLP作って」「HPBから7セクション埋めて」等のフレーズ
- 明示的に `salon-lp-builder` を指定

**起動しない**：HPB以外のサイトURL、サロン業種以外、SEO記事執筆依頼

## 前提依存

- **Playwright MCP**：`mcp__playwright__browser_navigate` `mcp__playwright__browser_snapshot` が利用可能であること
- **cvr-copywriting スキル**：同プロジェクト `.claude/skills/cvr-copywriting/` に存在すること

依存が満たされない場合、ユーザーに通知して中止する。

## 出力フォーマット（固定7セクション）

```
1. ファーストビュー（見出し / サブコピー / CTA / 補足）
2. 証明 / 実績 / 動画 / 信頼（見出し / 入れたい要素）
3. ベネフィット / 得られるもの（箇条書き）
4. こんな人におすすめ（箇条書き）
5. 流れ / ワークフロー / 違い（ステップ）
6. 登壇者 / 商品 / サービス紹介（名前 / 肩書き / プロフィール）
7. 最終CTA（CTA / 注記）
```

このフォーマットは固定。ユーザーから「セクション変更したい」と明示指示があった場合のみ変更可。

## 実行ステップ

### Step 1: URL受領＆事前確認

ユーザー提示URLが `beauty.hotpepper.jp` ドメインであることを確認。
それ以外なら「HPB限定スキルです。HPBの店舗ページURLを送ってください」と返して中止。

### Step 2: Playwright分析（自動実行）

```
1. mcp__playwright__browser_navigate(url=ユーザー提示URL)
2. mcp__playwright__browser_snapshot() でページ全体を取得
   ※snapshotが大きい場合はファイル保存→subagentで分析委譲
3. 以下の情報を抽出（verbatim quote 推奨）：
   - サロン名（日本語表記＋ローマ字）
   - 業種カテゴリ（ネイル / マツエク / 美容室 / エステ / リラク等）
   - 立地（駅徒歩分・住所）
   - 営業時間・定休日
   - 平均予約金額（初回 / 2回目以降）
   - クーポン価格（通常価格→特別価格、適用期間）
   - 口コミ集計：★平均評価・件数
   - リピート率・満足度（記載があれば）
   - スタッフ数・PICK UPスタッフ
   - 顧客属性比率（性別・年代）
   - 提供メニュー一覧
   - USP / こだわり条件（個室・完全予約制・女性専用・キャッシュレス等）
   - サロン側コピー文言の verbatim抜粋（コンセプト・売り）
   - 個別口コミ本文は数字集計用にのみ参照（個別文面は転載しない）
```

スナップショットが大きい場合（65,000文字超）は、ファイル保存→Exploreまたはgeneral-purpose subagentに以下指示で委譲：
> 「このsnapshotファイルから上記項目をverbatim抽出してください。1,138行を全て読んでから報告してください」

### Step 3: 主要ターゲットペルソナの確定

抽出した顧客属性比率から、最大シェアの属性を主ターゲット仮確定：
- 例：女性100% / 20代51% / 30代22% → 主：20代女性、副：30代女性
- これを cvr-copywriting Phase 1 の「仮置き設定ブロック」に反映

### Step 4: cvr-copywriting スキルの起動

`Skill` ツールで `cvr-copywriting` を呼び出し、以下の指示文を渡す：

```
HPB分析結果に基づき、フルコピーモードで以下の7セクション構成を埋めてください。
固定フォーマット遵守、3案軸分けではなく1本通しの完成稿を出力してください。

【分析データ】（Step 2 の抽出結果を全て貼り付け）

【7セクション構成】
（上記「出力フォーマット」をそのまま貼り付け）

【厳守事項】
- 薬機法・景表法遵守（「治る」「効く」「シミ消える」等NG）
- 口コミ個別文面は架空ペルソナのテンプレで提示（6件・属性多様化）
- ★評価・件数はHPB出典明記
- "No.1" "日本一" は使わない
- 抽象修飾語単独は禁止（数字との組み合わせのみ可）
- why now / why us / why you の三要素を必ず組み込む
```

### Step 5: 出力組み立て＆コピペ用整形

cvr-copywriting からの応答を受け、コードブロック1つに統合して**コピペ可能な形式**で出力する。

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ 1. ファーストビュー
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【見出し】
（FV見出し）

【サブコピー】
（サブコピー）

【CTA】
[ボタン]（CTA文言）

【補足】
（補足情報）


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ 2. 証明 / 実績 / 動画 / 信頼
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
（以下セクション7まで同形式で連結）
```

出力直後に1〜2行で：
- 採用フレーム名と主力心理トリガー
- 仮置き設定の概要（変更指示があれば再生成する旨）
を添える。

## 口コミセクションの扱い（重要ルール）

HPB掲載の個別口コミ本文は、**自社LPに無断転載すると規約・許諾上のリスクがある**ため：

- ★平均評価・件数（752件、★4.73）→ **「HPB調べ」と出典明記して利用OK**
- リピート率・満足度（サロン側公表値）→ **「{サロン名}公表値」明記で利用OK**
- 個別口コミ本文 → **架空ペルソナの仮置きテンプレ6件**を出力し、注記で「サロン側で実顧客から取材＋掲載許諾取得後に差し替え推奨」と明示

架空ペルソナテンプレ構造：
```
[N] {仮名}さん / {年齢}歳 / {職業}
　▼ 来店前の悩み：「{HPBから推測した顧客像の悩み}」
　▼ 選んだ理由：「{サロンUSPに対応する理由}」
　▼ 施術後：「{ベネフィットの具体描写}」
```

属性多様化のため、6件の年代・職業は分散させる（20代前半/20代後半/30代前半/30代後半/40代/職業も会社員/ワーママ/学生/専門職等で散らす）。

## Examples

### Example 1: HPBサロンURLからLP生成

User says:
> 「https://beauty.hotpepper.jp/kr/slnH000767465/ を分析してLP作って」

Actions:
1. URL確認 → HPBドメインOK
2. Playwright で navigate＋snapshot
3. snapshotが大きい場合はファイル保存→subagentで分析
4. 抽出データから主ターゲット仮確定（例：20代女性）
5. cvr-copywriting を Skill ツールで起動、抽出データ＋7セクション構成を渡す
6. 応答をコードブロック統合して出力

Result:
1〜7のセクションを統一トーンで埋めたコピペ可能なLPコピー。

### Example 2: 非HPB URLが送られた場合

User says:
> 「https://example-salon.com を分析して」

Actions:
1. ドメイン確認 → HPB外
2. 中止して「このスキルはHPB（beauty.hotpepper.jp）専用です」と返答

## Troubleshooting

### Error: Playwright snapshot が容量超過
**原因**：snapshotが65,000文字超でツール結果サイズ制限に抵触
**解決**：snapshotにファイル名指定で保存（`mcp__playwright__browser_snapshot(filename="snapshot.md")`）、その後 general-purpose subagent に該当ファイルパスと抽出項目リストを渡して委譲

### Error: cvr-copywriting スキルが見つからない
**原因**：プロジェクト `.claude/skills/cvr-copywriting/` が存在しない
**解決**：依存スキルなのでインストール要。ユーザーに通知して中止

### Error: 主要ターゲットが複数該当する
**原因**：HPB上の客層比率が分散している（例：20代30% / 30代30% / 40代25%）
**解決**：主ターゲット仮置きを宣言し、ユーザーに「20代寄り/30代寄り/両狙い」の選択を求める

### Warning: クーポン期限が過去日付
**原因**：HPB側で期限切れクーポンが残っている、または現在日付が期限後
**解決**：補足コピー内の期限部分を `{要更新：最新クーポン期限}` プレースホルダにし、ユーザーに通知

## 厳守ルール

- 出力は**固定7セクション**。勝手にセクション追加・削除しない
- 個別口コミ文面の**HPB転載は禁止**（数字集計は出典明記でOK）
- 薬機法・景表法違反表現は cvr-copywriting 側で検出されるが、本スキルでも最終チェック
- 「お任せ」依頼でも、Step 2 の Playwright 分析は**必ず実行**（HPBの最新クーポン日付・空席状況を反映するため）
- cvr-copywriting を呼ばずに自己流で7セクション埋めるのは禁止（参照宣言が出ない＝品質劣化のシグナル）
