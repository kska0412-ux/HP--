# ヘッドライン型カタログ

14サイトのFVから抽出した、実戦投入可能なヘッドラインパターン22種。
各パターンに「定義 / 穴埋めテンプレート / CVR改善効果タグ（高/中/低）/ 使用条件 / 失敗パターン / 実例典拠」を付す。

CVR改善効果タグの基準：
- **高**：単独で30%以上のCVR上昇が見込まれる強力型（FVキャッチ主役向き）
- **中**：他要素と組み合わせて10〜30%上昇が期待できる補助型
- **低**：単独効果は5%未満。サブコピー・補足配置向き

---

## 1. 数字インパクト型 【高】

### 定義
驚異性の高い数字を冒頭最大級で配置し、読者の視線を物理的に捕まえる。AIDAのA担当。

### テンプレート
```
驚異の{2桁以上の%／4桁以上の人数}！
{結果}が{実感率}％
{大数字}{単位}が{動詞}
```

### 使用条件
- ✅ 実証可能な数字を保有（実績・%・体験数）
- ✅ コールド〜ウォーム層流入のLP
- ❌ 数字の裏付けが弱い場合は景表法リスク

### 失敗パターン
- 数字を文章中に埋没させる（必ずデカ文字単独配置）
- 「驚異の」「圧倒的な」だけで数字を出し惜しみ

### 実例典拠
- `anka-tokushima`：「驚異の95%が実感」
- `seitai_sdk`：「87,540人以上が体験した」
- `sango_kotsuban`：「8,000人超／97%／98%」

## 2. 質問訴求型 【中】

### 定義
読者に問いを投げかけ、能動的に答えを探させることでエンゲージメントを起動。好奇心ギャップを利用する。

### テンプレート
```
{土地・業界}の{固有名詞}、知ってますか？
なぜ{ブランド}は{驚異の結果}を出せるのか？
{業界人}だけが知っている{秘密}とは
{あなた}は、{あるある状態}になっていませんか？
```

### 使用条件
- ✅ 独自手法・新カテゴリの商材
- ✅ 読者の知識ギャップが大きいテーマ
- ❌ 既知のジャンルで使うと「知ってるよ」と離脱

### 失敗パターン
- 答えがすぐ想像できる質問（読者の思考を止める）
- 中身を開けたら期待外れ（クリックベイト化）

### 実例典拠
- `anka-tokushima`：「徳島の施術知ってますか？」

## 3. 圧縮ストーリー型 【高】

### 定義
1行に過去のBefore・葛藤・解決・未来をすべて圧縮する。Hero's Journeyの極小バージョン。

### テンプレート
```
"{過去の状態}だった私"が、{現在の状態}を取り戻した／手に入れた理由
{挫折経験}した{主人公属性}の、{逆転}物語
```

### 使用条件
- ✅ 情緒コンバージョン商材
- ✅ Before/Afterが視覚化できる
- ❌ 機能・スペック比較が主軸のBtoB

### 失敗パターン
- 主人公属性が読者と乖離
- "取り戻した"の中身が抽象的

### 実例典拠
- `mito-hatsumou`：「"薬でも変わらなかった私"が、髪と心にふんわり 自信を取り戻した理由」

---

## 4. 制約付きYOUメッセージ型 【高】

### 定義
[制約条件]を含めて読者を絞り込み、特定状況の人だけを精密に呼び込む。QUEST のQ担当。

### テンプレート
```
{制約条件：資金がなくても／時間がなくても／一人でも／未経験でも}、
{願望：売れる新メニューを作りたい／理想のボディを手にいれる／…}
{役割：サロンオーナー／30代女性／…} へ
```

### 使用条件
- ✅ ニッチターゲット向けBtoB／高関与商材
- ✅ ターゲットの心理プロファイルが明確
- ❌ マスマーケット商材（読者を絞ると母数が減りすぎる）

### 失敗パターン
- 制約条件が読者の自己認知と一致しない
- 役割が広すぎて絞り込み効果が出ない

### 実例典拠
- `emdual`：「資金に余裕がなくても、売れる新メニューを作りたいサロンオーナー様へ」
- `revi-pro`：「もう、メニュー導入で失敗したくないサロンオーナーへ」
- `nobles`：「一人じゃ続かない…でも、諦めたくない！とお悩みのあなたへ」

## 5. カテゴリーキング否定型 【高】

### 定義
既存カテゴリを三段否定して、自社を独自カテゴリとして位置づける。

### テンプレート
```
{症状}、{"心"まで届いていませんか？}
{競合A}でも、{競合B}でも、{競合C}でもない
"{独自手法の象徴的動作}"なのに、{驚異の結果}
—— それが、{ブランド名・新カテゴリ名}
```

### 使用条件
- ✅ レッドオーシャン市場で差別化が必要
- ✅ コンセプトが独自で言語化できる
- ❌ コモディティ商品（USPが弱いと逆効果）

### 失敗パターン
- 否定する対象が読者の認知にない
- 「結局◯◯系では？」と思われる弱い独自性

### 実例典拠
- `otona-stretch`：「整体でも、マッサージでも、運動でもない／"触れるだけ"なのに、呼吸まで深くなる。—— それが、大人の脱力ストレッチ」

## 6. 結果先取り型 【高】

### 定義
ベネフィットの最終結果を時間付きで冒頭に置く。「いつまでに何が」を1行で言い切る。

### テンプレート
```
わずか{期間}で {結果数値}
{時間}後に {状態変化}
{N}日で {変化}が始まる
```

### 使用条件
- ✅ 結果が数値化できる商材（痩身・薄毛・整体）
- ✅ 期間と効果のセットを言い切れる事業者
- ❌ 効果に時間がかかる商材（半年・1年単位）

### 失敗パターン
- 期間を曖昧化（「すぐに」では作動しない）
- 結果が抽象的（「キレイになる」では数値化されない）

### 実例典拠
- `esthetic-sdk`：「わずか3ヶ月でトータル −42cm」
- `seitai_sdk`：「楽になる30分」

## 7. 共感問いかけ型 【中】

### 定義
読者の悩みを3点以上タグ化して並べ、「あなたのことを言っている」と認知させる。PASONA の P 担当。

### テンプレート
```
{悩み1}／{悩み2}／{悩み3} とお悩みのあなたへ
こんなことで困っていませんか？
　・{具体悩み1}
　・{具体悩み2}
　・{具体悩み3}
```

### 使用条件
- ✅ 悩み・痛み・コンプレックス商材
- ✅ ターゲットの悩み言語が明確
- ❌ 悩みが言語化されていない新カテゴリ

### 失敗パターン
- 悩みリストが抽象的すぎる
- 共感だけして次のステップを用意しない

### 実例典拠
- `sango_kotsuban`：悩みタグ4種を並列配置
- `anka-tokushima`：「どこへ行っても痛みが取れない…」
- `nobles`：「一人じゃ続かない…」

## 8. ベネフィット未来時制型 【中】

### 定義
読者の未来形を時制とともに描き、未来に時間旅行させる。フューチャーペーシング応用。

### テンプレート
```
{時間}後、{変化}が始まります
想像してください、{ライフスタイル}を
{固有時間}には、もう{達成状態}
```

### 使用条件
- ✅ Before/Afterが視覚化できる商材
- ✅ ライフスタイル変革を訴求する商材

### 失敗パターン
- 未来描写が抽象的すぎる
- 五感ディテールがない

### 実例典拠
- `mito-hatsumou`：「3ヶ月後、あなたの印象が変わり始めます」
- `koubo`：「私らしく前向きに」

## 9. 二択否定型 【中】

### 定義
「ただ◯◯するだけじゃ、◯◯」のパターンで、既存解の不足を指摘しながら自社の必要性を匂わせる。

### テンプレート
```
ただ{既存解}するだけじゃ、{ネガティブ結果}
{単純解A}か、{単純解B}か——もう選ばなくていい
{既存常識}の時代は、終わりました
```

### 使用条件
- ✅ レッドオーシャン市場の差別化
- ✅ 既存解への不満を持つ読者層

### 失敗パターン
- 否定の根拠が示されない
- 自社の代替案が見えない

### 実例典拠
- `theb`：「ただ痩せるだけじゃ、なんだか貧相になりそう」

## 10. 動詞インパクト型 【中】

### 定義
動詞単体に強烈な意味を持たせて2〜4文字で訴求。視覚的に余白を作り、印象を残す。

### テンプレート
```
{動詞2〜4文字：整える／脱力する／動く／キク／取り戻す}
```

### 使用条件
- ✅ ブランドコンセプトを象徴できる動詞がある場合
- ✅ デザイン主体のLPで余白が活かせる
- ❌ 動詞だけでは伝わらない複雑商材

### 失敗パターン
- 動詞が一般的すぎてブランド認知に貢献しない
- サブコピーで補わず投げっぱなし

### 実例典拠
- `theb`：「整える」
- `otona-stretch`：「脱力」
- `koubo`：「キク」（"効く"のひらがな化）

## 11. 数字×時間型 【高】

### 定義
「N分」「N秒」「N日」など短時間と結果を組み合わせ、即効性を訴求。

### テンプレート
```
{N}分で {結果}
{N}秒で {完了アクション}
たった{N}日で {状態変化}
```

### 使用条件
- ✅ 短時間で結果が出る商材
- ✅ モバイル流入が多いLP（短時間訴求に親和）

### 失敗パターン
- 実態と乖離した時間訴求（誇大広告リスク）

### 実例典拠
- `seitai_sdk`：「楽になる30分」
- `gfa202602`：「40分で疑問を解消」「簡単60秒で」
- `emdual`：「30分で結果が見える」「入力は1分」

## 12. 願望ハック型 【中】

### 定義
読者の本音の願望を「もう〜したくない」「諦めたくない」の形で言語化する。損失回避と共感の合成。

### テンプレート
```
もう、{過去の失敗経験}したくない{役割}へ
{願望}を、諦めたくないあなたへ
{現状}から、抜け出したいなら
```

### 使用条件
- ✅ 過去に同種商品で失敗経験のある層
- ✅ 内面葛藤が明確な悩み商材

### 失敗パターン
- 「諦めたくない」の中身が抽象的
- 読者の本音と微妙にズレる

### 実例典拠
- `revi-pro`：「もう、メニュー導入で失敗したくないサロンオーナーへ」
- `nobles`：「諦めたくない！」
- `lucil-academy`：「手に職をつけたいを諦めない」

## 13. ペルソナ呼びかけ型 【中】

### 定義
「{属性}の方へ」と明示的に名指す。広告と統合された場合の精度向上に効く。

### テンプレート
```
{属性：30代女性／サロンオーナー／産後ママ}の方へ
{役割}の、{固有悩み}を持つあなたへ
```

### 使用条件
- ✅ ペルソナが明確なBtoB／高関与商材
- ✅ 広告クリエイティブと連動するLP

### 失敗パターン
- ペルソナが広すぎて絞り込み効果が出ない
- 呼びかけの後に具体性が続かない

### 実例典拠
- `emdual`：「サロンオーナー様へ」
- `revi-pro`：「サロンオーナーへ」

## 14. ベネフィット羅列型 【低】

### 定義
ベネフィットを3〜5点リスト化して並べる。AIDA の I+D 担当。FVのサブブロック向き。

### テンプレート
```
✓ {ベネフィット1：時間圧縮}
✓ {ベネフィット2：コスト削減}
✓ {ベネフィット3：効果実感}
✓ {ベネフィット4：継続容易}
```

### 使用条件
- ✅ 複数機能訴求が必要な商材
- ✅ サブブロックでの補強訴求

### 失敗パターン
- 全項目が同じ抽象度
- チェックリストが長すぎる（5点以内に絞る）

### 実例典拠
- `emdual`：「30分で結果が見える／放置中に別業務／解約も自由」

## 15. 比喩・象徴型 【中】

### 定義
ベネフィットを別の事象に喩えて、価値の大きさを直感的に伝える。

### テンプレート
```
{ベネフィット}を、{大きな比喩物}に喩えるなら——
まるで、{象徴的シーン}のような
{大きな価値物}が手に入るとしたら？
```

### 使用条件
- ✅ ベネフィットが抽象的で訴求しづらい商材
- ✅ BtoB機器・サブスクなど価値の言語化が難しい場合

### 失敗パターン
- 比喩が読者の経験圏外
- 比喩が陳腐

### 実例典拠
- `emdual`：「スタッフをもう一人雇ったような稼働感」

## 16. 既存解否定型 【中】

### 定義

### テンプレート

```
{既存解A}では{不足}

```

### 使用条件

### 失敗パターン
- 競合名出しが直接すぎて法的リスク
- 否定の根拠が示されない

### 実例典拠

## 20. 数字バッジ三点セット 【高】

### 定義

### テンプレート

```
[累計実績 {N人}]  [満足度 {N%}]  [変化実感 {N%}]
```

### 使用条件

### 失敗パターン
- 3点中1点が弱い（最弱点に引っ張られる）
- 出典・期間が明示されない

### 実例典拠
- `sango_kotsuban`：「8,000人超／97%／98%」

## 22. 比較数値型 【中】

### 定義
自社と他社（または業界平均）の数値を並置して、優位性を視覚的に訴求する。

### テンプレート
```
他社平均 {数値X} ／ 当社 {数値Y}
業界相場 {価格X}円 ／ 当社 {価格Y}円
{機能項目}：他社×／当社○
```

### 使用条件
- ✅ 比較根拠がある商材
- ✅ 競合との明確な機能差がある場合
- ❌ 比較根拠が示せず景表法リスクがある場合

### 失敗パターン
- 出典・期間が明示されない
- 比較項目を自社有利に恣意選択

### 実例典拠
- `revi-pro`：「最小投資×最短スピード」（暗黙の業界比較）

## ステージ × 推奨ヘッドライン型 マッピング

読者のステージ理論（Eugene Schwartz）に対応した推奨型。

| ステージ | 読者の認知状態 | 推奨ヘッドライン型 | 補助型 |
|---|---|---|---|
| Unaware | 問題自体を認知していない | 質問訴求型 / 好奇心ギャップ | 動詞インパクト型 |
| Problem Aware | 問題は知っているが解決策を知らない | 共感問いかけ型 / 既存解否定型 | 願望ハック型 |
| Solution Aware | 解決策の存在を知っているが商品を知らない | 数字インパクト型 / カテゴリーキング否定型 | 比較数値型 |
| Product Aware | 商品を知っているが選ぶ理由がない | 制約付きYOU型 / 結果先取り型 | 数字バッジ三点セット |
| Most Aware | 商品の良さを知っているが買うきっかけがない | ジャストワード型 / 数字×時間型 | ペアコピー型 |

## ヘッドライン構築フロー

1. **ターゲットのステージを判定** → 上記マッピングから主役型を1つ選ぶ
2. **CVR効果タグ【高】の型を主役に配置** → 1つに絞り込む
3. **【中】【低】の型をサブブロックに配置** → 主役を補強する
4. **失敗パターンチェック** → 該当しないか1ヶずつ確認
5. **実例典拠と照らし合わせ** → 同型のヒット例があるか確認
