# CTAパターン辞典

14サイトのCTA分析から抽出した、文言・配置・色・サイズの定石とアンチパターン集。
CVRを最終的に決定する最重要要素。LPの他要素が完璧でもCTAで全てを台無しにできる。

---

## 1. CTA文言の定石

### 1-1. 関与度別の文言マトリクス

| 関与度 | 目的 | 推奨文言 | 摩擦感 |
|---|---|---|---|
| 超低関与 | LINE登録・SNSフォロー | 「LINEで友だち追加」「LINEで無料チェック」 | 1（ワンタップ） |
| 低関与 | 資料DL・メルマガ登録 | 「無料で資料を受け取る」「資料請求はこちら」 | 2（メアド入力） |
| 中関与 | 無料体験・無料診断 | 「無料体験を申し込む」「60秒の無料診断」 | 3（フォーム＋日程） |
| 高関与 | 説明会・個別相談 | 「無料で説明を聞いてみる」「個別相談を予約する」 | 4（日程調整＋電話or対面） |
| 最高関与 | 本契約・購入 | 「今すぐ申し込む」「購入する」 | 5（決済） |

### 1-2. 強い動詞の選び方

**強い動詞（推奨）**
- 「受け取る」（贈与イメージ・互恵性起動）
- 「申し込む」（具体的な行動）
- 「予約する」（時間枠の確保）
- 「始める」（新章の開始）
- 「試す」（リスクの低さ）

**弱い動詞（非推奨）**
- 「確認する」（受動的・行動感が弱い）
- 「送信」（読者の労力を強調）
- 「お問い合わせ」（敷居が高い印象）
- 「クリック」（具体性ゼロ）

### 1-3. 摩擦解消ラベル

CTAボタンの近辺に置く「不安解消」の小コピー。

```
※しつこい営業電話はありません
※入力は{秒数}で完了
※登録は無料です
※いつでも解約自由
※全額返金保証付き
```

### 実例典拠
- `emdual`：「フォームで問い合わせる」「LINEで問い合わせる」（チャネル分岐）＋「しつこい営業電話なし」
- `sango_kotsuban`：「LINEで無料チェック」
- `gfa202602`：「40分で疑問を解消／無料で説明を聞いてみる」＋「簡単60秒で／無料で資料を受け取る」
- `revi-pro`：「資料請求」
- `nobles`：「まずは60分の無料体験」

## 2. CTA配置の定石

### 2-1. 配置位置

| 位置 | 役割 | 必須度 |
|---|---|---|
| FV直下（ABOVE THE FOLD） | 即決層を即捕獲 | ★★★ 必須 |
| 中盤（信頼ブロック直後） | 社会的証明で温まった層を獲得 | ★★ 推奨 |
| 末尾（FAQ後） | 完走した最ホット層を獲得 | ★★★ 必須 |
| スマホ画面下固定（Sticky） | 全スクロール位置で常時露出 | ★★★ 必須（モバイル） |
| サイドバー固定（Sticky） | PCで全スクロール位置で常時露出 | ★ オプション |

### 2-2. Z字導線設計

読者の視線移動（左上→右上→左下→右下）に沿ってCTAを配置する。

```
[ロゴ]                          [サブCTA（軽め）]      ← Z字スタート
[FVキャッチ + 数字]
[ベネフィット説明]
[メインCTA（重め）]              ← Z字中盤
[社会的証明・実績]
[FAQ]
                                [最終CTA（最強オファー）]  ← Z字終端
```

### 2-3. 二段CTA配置

関与度の異なるリードを両方拾うために、高関与CTAと低関与CTAを併置する。

```
[メインCTA：オレンジ／緑]
「{時間}で疑問を解消 → {無料体験／説明会}を申し込む」

[サブCTA：青／グレー]
「{時間}で完了 → {資料請求／LINE登録}」
```

### 実例典拠
- `gfa202602`：「40分で疑問を解消／無料で説明を聞いてみる」＋「簡単60秒で／無料で資料を受け取る」（二段CTA）
- `emdual`：「フォームで問い合わせる」＋「LINEで問い合わせる」（チャネル分岐型二段CTA）

## 3. CTA色の定石

### 3-1. 色 × 関与度マッピング

| 色相 | 心理効果 | 推奨用途 |
|---|---|---|
| オレンジ / 赤 | 警戒・行動・緊急性 | メインCTA（高関与・即決系） |
| 緑 | 安心・GO・自然 | メインCTA（健康・治療系） |
| 青 | 信頼・知性・冷静 | サブCTA（資料DL・情報提供系） |
| ピンク / マゼンタ | 女性向け・優しさ | 女性向け美容・健康サービス |
| 黒 / 濃紺 | 高級・プレミアム | ハイエンドサービス・ハイブランド |
| 黄 | 注意喚起・警告 | サブブロック装飾（CTAには非推奨） |
| グレー | 控えめ・補助 | サブCTAの補助・キャンセル風 |

### 3-2. 補色対比の原則

CTAボタンは、その周辺背景色と**補色対比**となる色を選ぶ。背景が白基調なら濃色CTA、背景が濃色ならアクセントカラーCTA。

### 3-3. ホバー時の色変化

PCではマウスホバーで色をワントーン暗く／明るくする。タップ可能性を視覚的に伝える。

### 実例典拠
- `mito-hatsumou`：オレンジ系CTAで高関与訴求
- `koubo`：ピンク／マゼンタ系で女性層訴求
- `sango_kotsuban`：緑系LINE CTAで健康・安心訴求

## 4. CTAサイズの定石

### 4-1. 物理サイズ

- **モバイル**：ボタン高さ最低 48px、幅は画面幅の80%以上
- **PC**：ボタン高さ最低 56px、幅は最低 240px
- **タップターゲット**：親指1.5倍が目安（Apple HIG: 44px、Google: 48dp）

### 4-2. 文字サイズ

- ボタンラベル：最低 16px（モバイル）、18px〜24px（PC）
- 補助コピー（摩擦解消ラベル）：12〜14px
- ボタン内の主要動詞は太字

### 4-3. 余白

CTAボタンの上下に最低 40px の余白を確保する。詰め込むとボタンが目立たなくなる。

## 5. CTA構造テンプレート

### 5-1. 標準パターン

```
[アイコン：LINE/メール/書類]  [CTAラベル：強い動詞＋無料明示]
　　　　　　　　　　           ※摩擦解消ラベル（しつこい営業電話なし etc.）
```

### 5-2. オファー強化パターン

```
[割引バッジ：今だけ50%OFF]
[アイコン]  [CTAラベル：通常価格→特別価格を明示]
　　　　　   ★先着{N}名様限定 / ★全額返金保証付き
```

### 5-3. 二段CTAパターン

```
[メインCTAボタン：高関与・大サイズ・暖色]
  動詞：「申し込む」「予約する」「始める」

[サブCTAボタン：低関与・中サイズ・寒色]
  動詞：「受け取る」「ダウンロードする」「友だち追加する」
```

## 6. アンチパターン10選

### NG 1. CTAが目立たない
- 背景色との対比が弱い／グレー基調
- サイズが小さい
- 余白が不足
- 解決：補色対比＋十分なサイズ＋上下40px余白

### NG 2. 文言が抽象的
- 「お問い合わせはこちら」「詳しくはこちら」
- 何が起きるか不明で行動コストが見えない
- 解決：「{秒数}で完了」「無料で{価値物}を受け取る」と具体化

### NG 3. CTAが多すぎる
- FVに5個並ぶ／中盤で3種の異なるCTA
- 認知負荷で読者がフリーズ
- 解決：主役CTAは1つ、補助は最大1つの二段構造

### NG 4. ボタンに見えない
- 角丸なし／影なし／枠線だけ
- リンクテキストにしか見えない
- 解決：背景色＋角丸＋ホバー効果でボタン認識を強化

### NG 5. 重要情報をCTA前で出し切る
- FVですべてのオファーを開陳
- 終盤CTAに押し材料が残らない（ピークエンドの法則違反）
- 解決：最強オファーは終盤CTAに温存

### NG 6. 摩擦解消ラベルの欠如
- 「営業電話あるかも」「解約大変かも」が解消されない
- 解決：CTAボタン直下に「※しつこい営業電話なし／いつでも解約自由」

### NG 7. CTA前の心理的負債
- 「以下のフォームをご記入の上、送信ボタンを押してください」
- 労力を強調するコピーで読者を萎えさせる
- 解決：「{秒数}で完了の入力フォーム」と短さを強調

### NG 8. モバイル固定CTAの欠落
- スクロール中にCTAが視界から消える
- 解決：画面下にスマホ用Sticky CTAを実装

### NG 9. リンク色との混同
- 青地白文字ボタンが「通常のリンク」に見える
- 解決：オレンジ／緑などの非リンク色を使う

### NG 10. CTA直前の離脱誘発要素
- 「他社比較表」「料金詳細」など考えさせる要素をCTA直前に置く
- 解決：CTA直前は感情のピーク（社会的証明・最強オファー）で締める

## 7. CTA設計チェックリスト

LP公開前に必ず通すチェック項目。

### 文言
- [ ] 動詞が「受け取る／申し込む／予約する／始める／試す」など能動的か
- [ ] 「無料」「{秒数}で」など具体的な情報が含まれているか
- [ ] 「お問い合わせはこちら」など抽象表現になっていないか
- [ ] 摩擦解消ラベル（営業電話なし／解約自由）がCTA直下にあるか

### 配置
- [ ] FV直下にCTAがあるか
- [ ] 中盤（信頼ブロック直後）にCTAがあるか
- [ ] 末尾（FAQ後）にCTAがあるか
- [ ] モバイル用Sticky CTAが実装されているか

### 色・サイズ
- [ ] 背景と補色対比になっているか
- [ ] モバイルでボタン高さ48px以上、幅80%以上か
- [ ] PCでボタン高さ56px以上、幅240px以上か
- [ ] 上下に40px以上の余白があるか
- [ ] ホバー効果が設定されているか（PC）

### 二段CTA
- [ ] 高関与CTAと低関与CTAが併置されているか
- [ ] 二段それぞれの動詞が適切に分かれているか
- [ ] 色相で関与度を視覚的に分けているか

### 最終
- [ ] 末尾CTAに最強オファー（割引＋限定＋保証）が集約されているか
- [ ] CTA直前が感情ピーク（社会的証明・物語完結）で締められているか
- [ ] CTAをタップした後の動線（フォーム／LINE）も同基準で設計されているか
