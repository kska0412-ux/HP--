# CVR Copywriting Skill

CVR最大化のための、コピー生成・添削・A/Bテスト案出しの構造化Skill。
14LP分析から抽出したナレッジ（`knowledge/` 6ファイル）と媒体別制約（`references/`）を**条件付き選択読み込み**で参照する。

---

## 起動条件

以下に該当する依頼で起動する：

- コピー生成（LP・広告・メルマガ・CTA・SNS・PR）
- 既存コピーの添削・改善
- A/Bテスト案出し
- FV設計・購入障壁の特定

**起動しない**：誤字脱字校正のみ／ブランディング／SEO記事執筆（CVR最適化が主目的でない場合）

---

## Phase 1: ヒアリング（不足項目のみ）

確認項目（揃っているものはスキップ）：

1. **商材**：何を売るか／単価／業種
2. **ターゲット**：年齢・性別・悩み・既存解への失敗
3. **媒体**：LP-FV／LP中盤／LP末尾／Meta／Google検索／メルマガ件名／LINE／バナー／SNS／PR
4. **現状コピー**：原文／現状CVR
5. **KPI**：CVR目標／客単価／LTV

### 原則
- 1ターンで聞くのは最大3問
- 「お任せ」or 部分回答時は商材から推測し、**仮置き宣言ブロック**を明示してから進む
- 既存コピーがあれば最初に読んでから不足を聞く

### 仮置き宣言ブロック

```
【仮置き設定（変更点があればお知らせください）】
- 商材詳細：{推測値}
- ターゲット悩み：{推測値}
- 単価帯：{推測値}
- 媒体・配置：{推測値}
- KPI目標：{推測値}
```

仮置きは `target-personas.md` のペルソナ＋ `frameworks.md` の業種カテゴリから類推。

## Phase 3: 生成

**3案以上**（指定があればその数）。フルコピーモード／パーツ単体モードを依頼内容で使い分ける。

### フルコピーモード（FV／LPセクション／広告クリエ一式）

```
## 案 {N}（軸ラベル）：{特徴1行}

### コピー本文
{ヘッドライン}
{サブコピー}
{ベネフィット部}
{CTA}

### 設計根拠
- 採用フレーム：{frameworks.md 参照}（主力／補助）
- 心理トリガー（3つ以上）：{1}／{2}／{3}
- ヘッドライン型：{headline-patterns.md 参照}【効果：高/中/低】
- ターゲットペルソナ：{target-personas.md 参照}
- 想定インパクト：{どう刺さるか}

### A/Bテスト時の検証ポイント
- 仮説：{上がる前提}
- 計測指標：CVR以外（CTR/直帰率/フォーム到達率など）
- 対立案との差分：{主要変数}

### 自己採点（合計/50）
- 具体性 {N}/10：{根拠1行}
- ベネフィット明確さ {N}/10：{根拠}
- ターゲット一致度 {N}/10：{根拠}
- 心理トリガー強度 {N}/10：{根拠}
- 行動喚起の強さ {N}/10：{根拠}
```

### パーツ単体モード（CTA／件名／ヘッドラインのみ）

```
## 案 {N}（軸ラベル）：{特徴1行} （{文字数}字 / 上限{媒体別上限}字）

### 本文
{パーツ本文}
{摩擦解消ラベル or プレヘッダー or 補助コピー}

### 設計根拠
- 心理トリガー（2つ以上）：{1}／{2}
- 採用パターン：{headline-patterns.md or cta-patterns.md 参照}
- 想定インパクト：{1〜2行}

### 自己採点（合計/30）
- 具体性 {N}/10：{根拠}
- 心理トリガー強度 {N}/10：{根拠}
- 行動喚起の強さ {N}/10：{根拠}
```

### N案軸分け原則

**同じフレームの語彙違いはNG**。冒頭ラベルに軸名を必ず明示。

| 案 | 軸 | 主な手法 |
|---|---|---|
| A | 感情訴求 | PASONA／ストーリー型 |
| B | 論理訴求 | QUEST／PREP |
| C | オファー訴求 | オファー先行／カテゴリーキング |
| D | 互恵訴求 | 無料診断・チェックリスト |
| E | マイクロコミット | LINE・60秒チェック |
| F | ペルソナシフト | 隣接ペルソナ狙い |
| G | 媒体トーン | 同商材を別媒体トーン翻訳 |

## Phase 4: 自己採点

採点軸・基準・採点後アクションは `references/scoring-rubric.md` を参照（必要時のみ Read）。

要点：
- フルコピー：5軸50点満点／40点以上で本番候補
- パーツ単体：3軸30点満点／24点以上で本番候補
- 各軸の**点数の右に1行で根拠**を必ず添える（甘い採点防止）

---

## 厳守ルール

### 表現規制
- **薬機法・景表法違反は禁止**
  - 医療：「治る」「効く」「改善」（医薬品でない場合）
  - 美容：「シミが消える」「シワがなくなる」
  - 健食：「ダイエット効果」「病気予防」
- **「No.1／日本一／世界初」は出典確認を促す**
- **抽象修飾語は禁止**：「最高の」「素晴らしい」「圧倒的な」「驚きの」単独はNG → 数字との組み合わせのみ可

### 構造ルール（1コピー必須3要素）
- **why now**：希少性／緊急性／損失回避 のいずれか1つ以上
- **why us**：独自性／実績数字／権威性／カテゴリーキング のいずれか1つ
- **why you**：制約付きYOU／ペルソナ呼びかけ／共感問いかけ のいずれか1つ

## モード別追加フロー

### 添削モード（「直して」「添削して」「改善案」）
1. 原文を `cta-patterns.md` アンチパターン10と照合
2. 原文を `objection-handling.md` 障壁トップ10と照合（漏れ検出）
3. 原文の語彙を `target-personas.md` 語彙辞典と照合
4. 修正案を**最低3案**生成（Phase 3 テンプレ）
5. 各案で「原文 vs 修正案」差分と「起動させた心理／フレーム」を1行明示
6. 原文の良かった点を最低1つ言及

### A/Bテストモード（「A/Bテストの案を」）
1. 検証仮説を1つに絞る（複数変数を同時に変えない）
2. 対立2案を生成（A=現状寄り／B=仮説検証用）
3. 変数の差分を1行明示（「ヘッドライン型のみ」等）
4. 計測指標を提示（CVR以外も：CTR・スクロール率・フォーム到達率）
5. 想定サンプルサイズの目安を提示

## 出力ルール

- 日本語／コピー本文はコードブロック
- ナレッジ参照箇所はファイル名を明示（例：「`frameworks.md` PASONA を採用」）
- 1ターンで3案＋根拠＋採点まで全部出す
- 「もっと尖らせて」等の方向指定があれば該当軸を強化した追加案を出す

---

## 失敗パターン

- ヒアリング5項目を一度に聞く → 質問疲れ
- ナレッジ6ファイル全部読む → context 浪費（Phase 2 判断表で必要分のみ）
- 「参照した」と主張するが自己流生成 → 出力冒頭の参照宣言で防ぐ
- 3案が同じフレームの語彙違い → 軸分け原則
- 採点が全項目8点以上で甘い → 採点根拠1行ずつ必須
- why now を組み込み忘れ → 構造ルールで強制
- 抽象修飾語の混入 → 表現規制で検出
- 薬機法・景表法表現 → 表現規制で検出
