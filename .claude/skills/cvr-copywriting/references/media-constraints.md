# 媒体別制約リファレンス

媒体ごとに文字数制約・配置構造・摩擦特性が異なる。Phase 3 生成時に該当媒体のセクションのみ読み込むこと。

---

## LP-FV（ファーストビュー）

- 文字数制約：1スクロール内で完結／メインキャッチ20〜40字／サブコピー60〜100字
- 構造：メインキャッチ + サブ + ベネフィット3点 + 第1CTA（ボタン文言は10〜15字）
- 摩擦特性：流入直後・3〜5秒で離脱判断・広告流入は特に冷たい
- 必須要素：数字 or 期間 or 固有名詞／CTAボタン直下に摩擦解消ラベル／why now
- 参照ナレッジ：`headline-patterns.md`（型選定）／`cta-patterns.md`（FV-CTA文言・配置）／`target-personas.md`（語彙整合）

## LP中盤

- 文字数制約：各セクション本文200〜400字／見出し15〜25字
- 構造：共感ブロック → 課題深掘 → 解決提示 → 社会的証明（実績数字／お客様の声） → 価格 → FAQ
- 摩擦特性：関心は持ったが疑い段階・「自分には合わないかも」「もっと安いのがある」を払拭する場
- 必須要素：障壁潰し（`objection-handling.md` 上位5障壁を順に潰す）／3つ以上の実績数字／第三者証明1つ以上
- 参照ナレッジ：`objection-handling.md`／`psychological-triggers.md`／`frameworks.md`（PASONA／QUEST）

## LP末尾

- 文字数制約：最終CTA周辺見出し20字／保証文80〜150字／希少性訴求30〜60字
- 構造：最後の一押し（リスク反転） → 保証 → 希少性／緊急性 → 最終CTA再提示
- 摩擦特性：決断直前のためらい・「もう少し考えたい」「あとで」を断ち切る場
- 必須要素：why now（先着N名／期間限定／放置リスク）／保証文言／強動詞CTA／損失回避トリガー
- 参照ナレッジ：`cta-patterns.md`／`psychological-triggers.md`（損失回避・希少性・緊急性）

## Meta広告（Facebook / Instagram）

- 文字数制約：本文125字以内推奨（「もっと見る」で切れる）／見出し40字／説明30字
- 構造：1行目フック → ベネフィット → 行動喚起
- 摩擦特性：スクロール中・受動的・3秒で離脱判断
- 必須要素：1行目に数字 or 質問でスクロール停止／emoji 1〜2個まで
- 参照ナレッジ：`headline-patterns.md`／`psychological-triggers.md`（好奇心・社会的証明）／`target-personas.md`

## Google検索広告

- 文字数制約：見出し30字×15本／説明文90字×4本
- 構造：検索意図直結のキーワード×3 + ベネフィット + 価格訴求
- 摩擦特性：能動的検索層・比較検討モード
- 必須要素：検索クエリの語を見出しに含める／価格 or 数字 or 期間を含める
- 参照ナレッジ：`headline-patterns.md`（数字インパクト型・カテゴリーキング型）／`frameworks.md`（QUEST）

## メルマガ件名＋プレヘッダー

- 文字数制約：件名PC全角30字／モバイル20字（最初の20字で勝負）／プレヘッダー30〜50字
- 構造：件名 + プレヘッダー（プレビュー欄）の2層セット設計
- 摩擦特性：受信トレイ内・他社件名との並列比較・3秒以下で開封判断
- 必須要素：why now／件名の冒頭20字に主要訴求／プレヘッダーは件名で言い切れなかった補足
- 参照ナレッジ：`headline-patterns.md`／`psychological-triggers.md`（好奇心・損失回避・希少性）

## LINEメッセージ

- 文字数制約：通知に出る最初の20〜30字が勝負
- 構造：冒頭フック → 本文 → CTAボタン
- 摩擦特性：プッシュ通知・即時性高・スワイプで瞬殺リスク
- 必須要素：絵文字1〜2個で視認性向上／リッチメニュー・カードと連動
- 参照ナレッジ：`cta-patterns.md`（マイクロコミットCTA）／`headline-patterns.md`

## バナー広告

- 文字数制約：最大15字程度（視認1秒）
- 構造：キャッチ1行 + 数字 + CTAボタン
- 摩擦特性：ディスプレイ面の中・無関心層・0.5秒で判断
- 必須要素：数字インパクト型ヘッドライン1択／ロゴ・色で記憶残存
- 参照ナレッジ：`headline-patterns.md`（数字インパクト型・問いかけ型）

## SNS投稿（X / Threads）

- 文字数制約：X=140字（日本語）／Threads=500字
- 構造：1行目フック → 本文 → CTA・リンク
- 摩擦特性：タイムライン中・流し読み・1〜2秒で判断
- 必須要素：1行目に質問 or 数字 or 反語
- 参照ナレッジ：`headline-patterns.md`（問いかけ型・反語型）／`psychological-triggers.md`（好奇心ギャップ）

## プレスリリース

- 文字数制約：タイトル30字以内推奨／リード文200字
- 構造：タイトル → 5W1Hで圧縮したリード → 詳細 → 会社情報
- 摩擦特性：記者は1日100本以上を流し読み・編集判断は数秒
- 必須要素：「日本初」「業界初」等の見出しフック（根拠必須）／数字・固有名詞・期間
- 参照ナレッジ：`frameworks.md`（PREP／結論先行）／`headline-patterns.md`（カテゴリーキング型・新規性型）
